Overview

What is Angular Material?

📖 Definition

Angular Material is the official UI component library for Angular, built by the Angular team at Google. It implements Google's Material Design specification — a visual language that combines classic principles of good design with innovation of technology.

🌐 Official Resource

Website: material.angular.dev

GitHub: github.com/angular/components

Package: @angular/material

✅ Key Qualities

  • High Quality — Internationalized (i18n) and accessible (a11y) by default
  • Versatile — Customizable within Material Design bounds; tools to build your own components
  • Frictionless — Built by Angular team; integrates seamlessly with Angular ecosystem
  • Well Tested — Each component has component harnesses for reliable testing

📦 What's Included

  • 60+ UI components — forms, navigation, layout, data display, overlays
  • Angular CDK — Component Dev Kit for building custom components
  • Theming system — Design token-based custom theming
  • Schematics — Auto-generate views with Material components

Setup

Installation & Setup

⚡ Install via Schematic (Recommended)

Use the Angular CLI schematic to automatically configure everything:

# Installs + configures Angular Material automatically
ng add @angular/material

# Prompts you to choose:
# 1. Pre-built theme (Azure/Blue, Rose/Red,
#    Cyan/Orange, Purple/Green)
# 2. Set up global typography styles?
# 3. Set up browser animations?

📦 Manual Installation

# Step 1: Install packages
npm install @angular/material @angular/cdk

# Step 2: Add pre-built theme in styles.scss
@use '@angular/material' as mat;

# Step 3: Import BrowserAnimationsModule
import { BrowserAnimationsModule }
  from '@angular/platform-browser/animations';

📁 What Gets Auto-Configured

  • Theme CSS added to angular.json styles array
  • BrowserAnimationsModule added to app.module.ts
  • Material icons font added to index.html
  • Typography styles added to styles.scss

🧩 Importing Individual Modules

Each component has its own module — import only what you use:

// In your component or shared module
import { MatButtonModule }
  from '@angular/material/button';
import { MatInputModule }
  from '@angular/material/input';
import { MatTableModule }
  from '@angular/material/table';

// Standalone (Angular 14+)
@Component({
  imports: [MatButtonModule, MatInputModule]
})
Best Practice

Create a SharedMaterialModule that imports and re-exports all Material modules your app uses — avoids repetitive imports.

Theming

Theming System

🎨 Material 3 Theming (Design Tokens)

Angular Material uses a SASS-based theming system powered by design tokens. Define a custom theme with primary, accent, and warn palettes.

// styles.scss
@use '@angular/material' as mat;

// Include core styles once
@include mat.core();

// Define your theme
$my-theme: mat.define-theme((
  color: (
    theme-type: dark,
    primary: mat.$violet-palette,
    tertiary: mat.$orange-palette,
  ),
  typography: (
    brand-family: 'Roboto',
  ),
  density: (
    scale: 0
  )
));

// Apply the theme
html {
  @include mat.all-component-themes($my-theme);
}

🔠 Typography

Material typography config defines font families and sizes for all text levels:

Dark Mode

Set theme-type: dark in the theme config or use @include mat.all-component-color-themes($dark-theme) scoped to a CSS class for toggle.

Material 2 vs Material 3

Angular Material supports both M2 (Material 2) using mat.define-light-theme() and M3 (Material 3) using mat.define-theme(). M3 is the current recommended approach with design tokens for better customization.

Data Display

mat-table — Data Table

📊 mat-table (CDK-Based)

Angular Material table is built on top of CDK data table — uses a declarative column definition pattern with powerful sorting, pagination, and filtering.

// Component
displayedColumns = [
  'name', 'status', 'lastService'
];
dataSource = new MatTableDataSource(
  this.equipmentList
);

ngAfterViewInit() {
  this.dataSource.sort =
    this.sort;
  this.dataSource.paginator =
    this.paginator;
}

// Filter
applyFilter(event: Event) {
  const val = (event.target
    as HTMLInputElement).value;
  this.dataSource.filter =
    val.trim().toLowerCase();
}
<!-- Template -->
<mat-form-field>
  <input matInput
    (keyup)="applyFilter($event)"
    placeholder="Search...">
</mat-form-field>

<table mat-table
       [dataSource]="dataSource"
       matSort>

  <!-- Column Definition -->
  <ng-container
    matColumnDef="name">
    <th mat-header-cell
        *matHeaderCellDef
        mat-sort-header>
      Name
    </th>
    <td mat-cell
        *matCellDef="let row">
      {{row.name}}
    </td>
  </ng-container>

  <tr mat-header-row
      *matHeaderRowDef="displayedColumns">
  </tr>
  <tr mat-row
      *matRowDef="let row; columns: displayedColumns">
  </tr>
</table>

<mat-paginator
  [pageSizeOptions]="[5,10,25]"
  showFirstLastButtons>
</mat-paginator>
MatTableDataSource Powers
  • Built-in client-side sorting via matSort
  • Built-in client-side filtering via dataSource.filter
  • Built-in pagination via MatPaginator
  • For server-side: implement custom DataSource extending DataSource<T>

constructor(private dialog: MatDialog) {}

openDialog() {
  const ref = this.dialog.open(

    width: '400px',
    data: { title: 'Confirm' }
  });

    result => {
      if (result) this.doAction();

}

// In dialog component
constructor(

) {}

CDK

Angular CDK — Component Dev Kit

🛠️ What is CDK?

CDK = Component Dev Kit. It is the foundation layer of Angular Material — a set of behaviour primitives that help you build your own custom components without being tied to Material Design's visual style.

🖐️ Drag & Drop

Full drag-and-drop with reorder support:

<div cdkDropList
  (cdkDropListDropped)="drop($event)">
  <div *ngFor="let item of list"
       cdkDrag>
    {{item}}
  </div>
</div>

Interview

Interview Questions & Answers

Q1. What is the difference between mat-button, mat-raised-button, mat-flat-button, and mat-stroked-button?
mat-button — text-only, no background/border. mat-raised-button — elevated with shadow. mat-flat-button — filled background, no shadow. mat-stroked-button — outlined border, transparent background. All accept color="primary|accent|warn".
Q2. What is MatTableDataSource and what does it provide out of the box?
MatTableDataSource is a data source wrapper that provides client-side filtering (via .filter property), sorting (connect a MatSort), and pagination (connect a MatPaginator). For server-side operations, implement a custom class extending DataSource<T>.
Q3. How does Angular Material theming work?
Angular Material uses a SASS-based theming system. You define a theme using mat.define-theme() with primary/accent/warn palettes and include it via mat.all-component-themes($theme). M3 (Material 3) uses design tokens. Multiple themes can be scoped to CSS classes for dark mode support.
Q4. What is the difference between Angular Material and Angular CDK?
Angular CDK provides behaviour primitives (overlay, drag-drop, virtual scroll, a11y) with no visual styling. Angular Material uses CDK internally and adds Material Design visual styles. You use CDK when building custom components that need these behaviours without Material Design's look.
Q5. How do you pass data to and receive data from MatDialog?
Pass data via the data property in dialog.open(Component, { data: {...} }). Inject it in the dialog component using @Inject(MAT_DIALOG_DATA). Receive result via dialogRef.afterClosed().subscribe(result => ...). Close with data using dialogRef.close(returnValue).
Q6. What are the three sidenav modes and when would you use each?
over — sidenav floats over content (mobile, temporary). push — sidenav pushes content sideways (tablet). side — sidenav sits beside content permanently (desktop). Typically combine all three with responsive breakpoints using BreakpointObserver from CDK.
