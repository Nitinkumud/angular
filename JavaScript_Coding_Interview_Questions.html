Q1 — Implement a Closure-based Counter Factory

The Question

Write a createCounter() function that returns an object with increment, decrement, and getCount methods. Each counter instance must have its own private state.

function createCounter(initialValue = 0) {
  let count = initialValue; // private via closure

  return {
    increment() { count++; },
    decrement() { count--; },
    getCount()  { return count; },
    reset()     { count = initialValue; }
  };
}

const c1 = createCounter(10);
const c2 = createCounter(0);

c1.increment(); c1.increment();
c2.decrement();

console.log(c1.getCount()); // 12
console.log(c2.getCount()); // -1 — independent!

Why Closures Work Here

  • Each call to createCounter creates a new execution context
  • Returned methods close over that specific count variable
  • count is not accessible from outside — true encapsulation

Classic Gotcha — Loop Closure

// BUG — all print 5
for (var i=0; i<5; i++)
  setTimeout(()=>console.log(i), 0);

// FIX — use let (block scope)
for (let i=0; i<5; i++)
  setTimeout(()=>console.log(i), 0);

⚡ Angular link: RxJS operators like scan and service-level state management use the same closure-over-state principle under the hood.

Temporal Dead Zone (TDZ)

let and const bindings exist in scope from the start of the block but accessing them before the declaration line throws ReferenceError. This is the TDZ.

Best Practice

Always use const by default, let when reassignment is needed. Never var in modern code.

Q3 — Event Loop: Predict the Exact Output

The Question

What is the exact output of the code below? Explain the role of the Call Stack, Microtask Queue (Promises), and Macrotask Queue (setTimeout).

console.log('1 - sync');

setTimeout(() => console.log('2 - timeout'), 0);

Promise.resolve()
  .then(() => console.log('3 - promise then'))
  .then(() => console.log('4 - promise then 2'));

queueMicrotask(() => console.log('5 - microtask'));

console.log('6 - sync');
// Output (in order)

3 - promise then
5 - microtask
4 - promise then 2

Execution Order Rules

  • Synchronous code runs first (Call Stack)
  • Microtasks drain completely before any macrotask (.then, queueMicrotask, MutationObserver)
  • Each resolved .then schedules the next .then as a new microtask
  • Macrotasks run one per event loop tick (setTimeout, setInterval, I/O)

Q4 — Promise Chaining & Error Handling

The Question

Fix the broken promise chain below. Then add error handling so that a single .catch at the end handles all errors, but step 2 can recover and continue.

// ❌ Broken — not chained, errors swallowed
const p = fetchUser(1);
p.then(user => fetchOrders(user.id));
p.then(orders => console.log(orders));

// ✅ Fixed — properly chained
fetchUser(1)
  .then(user => fetchOrders(user.id))
  .then(orders => processOrders(orders))
  .then(result => console.log('Done:', result))
  .catch(err => console.error('Error:', err))
  .finally(() => console.log('Cleanup'));

// Recovery in middle of chain
fetchUser(1)
  .then(user => fetchOrders(user.id))
  .catch(() => []) // step 2 fails → recover with []
  .then(orders => console.log(orders)) // always runs
  .catch(err => console.error(err));

Key Rules

  • .then() returns a new Promise — always chain on the returned value
  • A .catch() in the middle recovers the chain — subsequent .then() runs
  • Throwing inside .then() passes error to next .catch()
  • .finally() runs always, receives no value, does not affect chain

Promise States

  • Pending → initial state
  • Fulfilled → value resolved
  • Rejected → reason available
  • State transitions are irreversible

⚡ Angular: HttpClient returns Observables but internally wraps Promises. Converting with firstValueFrom() needs a try/catch.

Q5 — async/await: Parallel vs Sequential

The Question

The code below takes 4 seconds. Refactor it so both API calls run in parallel (total ~2 seconds). Also handle errors without crashing the whole function.

// ❌ Sequential — 4s total
async function loadDashboard() {
  const user   = await fetchUser();   // 2s
  const orders = await fetchOrders(); // 2s
  return { user, orders };
}

// ✅ Parallel — ~2s total
async function loadDashboard() {
  const [user, orders] = await Promise.all([
    fetchUser(),
    fetchOrders()
  ]);
  return { user, orders };
}

// ✅ Parallel + individual error handling
async function loadDashboardSafe() {
  const safe = p => p.catch(e => ({ error: e.message }));
  const [user, orders] = await Promise.all([
    safe(fetchUser()),
    safe(fetchOrders())
  ]);
  return { user, orders };
}
MethodBehavior on Reject
Promise.all()Short-circuits on first rejection
Promise.allSettled()Always waits for all, reports each status
Promise.race()Resolves/rejects with first to settle
Promise.any()Resolves with first fulfilled; rejects only if all reject

async/await is Syntactic Sugar

An async function always returns a Promise. await pauses execution of that function only — the event loop continues.

⚡ Angular: Prefer forkJoin() (RxJS) over Promise.all, but understand the underlying Promise semantics for debugging and testing.

Q7 — The this Keyword: 4 Binding Rules

The Question

What are the 4 rules that determine the value of this? What does this refer to in arrow functions vs regular functions?

// 1. Default binding (global / undefined in strict mode)
function show() { console.log(this); } // window / undefined

class Timer {
  constructor() { this.ticks = 0; }
  start() {
    // ❌ regular fn — this is undefined in strict
    setInterval(function() { this.ticks++; }, 1000);
    // ✅ arrow fn — this captured from start()
    setInterval(() => { this.ticks++; }, 1000);
  }
}

Arrow Function Rule

Arrow functions have no own this. They inherit this from the enclosing lexical scope. Cannot be used as constructors. bind/call/apply cannot change their this.

⚡ Angular: Class methods in components used as event handlers (e.g., (click)="handleClick") are called with this of the component — Arrow functions or bind needed only in imperative DOM code.

Function.prototype.myCall = function(context, ...args) {
  context = context ?? globalThis;
  const sym = Symbol(); // unique key to avoid collisions
  context[sym] = this;  // attach fn as method of context
  const result = context[sym](...args);
  delete context[sym];   // clean up
  return result;
};

Function.prototype.myApply = function(context, argsArr = []) {
  return this.myCall(context, ...argsArr);
};

Function.prototype.myBind = function(context, ...preset) {
  const fn = this;
  return function(...args) {
    return fn.myCall(context, ...preset, ...args);
  };
};

Q9 — Implement Debounce & Throttle

The Question

Implement debounce(fn, delay) and throttle(fn, limit). Explain the difference and when to use each.

// DEBOUNCE — fires AFTER user stops (last call wins)
function debounce(fn, delay) {
  let timer;
  return function(...args) {
    clearTimeout(timer);
    timer = setTimeout(() => fn.apply(this, args), delay);
  };
}

// THROTTLE — fires AT MOST once per limit period
function throttle(fn, limit) {
  let lastCall = 0;
  return function(...args) {
    const now = Date.now();
    if (now - lastCall >= limit) {
      lastCall = now;
      return fn.apply(this, args);
    }
  };
}

// Usage examples
const debouncedSearch = debounce(search, 300);
input.addEventListener('input', debouncedSearch);

const throttledScroll = throttle(onScroll, 100);
window.addEventListener('scroll', throttledScroll);
DebounceThrottle
Fires whenAfter silenceAt regular intervals
Best forSearch input, resize-endScroll, mousemove, rate-limiting
BehaviourResets timer on each callIgnores calls within window

Leading vs Trailing Edge

Production debounce implementations support { leading: true } to fire immediately on the first call, then debounce subsequent calls. Libraries like Lodash expose these options.

⚡ Angular: Use RxJS debounceTime(300) and throttleTime(100) operators on form control valueChanges observables for the same effect declaratively.

Q10 — Implement a Generic Curry Function

The Question

Write a curry(fn) function that transforms any multi-argument function so it can be called with one argument at a time. Support both curried and multi-arg invocations.

function curry(fn) {
  return function curried(...args) {
    if (args.length >= fn.length) {
      // Enough args — call original fn
      return fn.apply(this, args);
    }
    // Not enough — return function collecting more
    return function(...moreArgs) {
      return curried.apply(this, [...args, ...moreArgs]);
    };
  };
}

// Example
const add = (a, b, c) => a + b + c;
const curriedAdd = curry(add);

curriedAdd(1)(2)(3);   // 6
curriedAdd(1, 2)(3);  // 6
curriedAdd(1)(2, 3);  // 6
curriedAdd(1, 2, 3); // 6

// Real-world: reusable pipelines
const multiply = curry((a, b) => a * b);
const triple   = multiply(3);
[1, 2, 3].map(triple); // [3, 6, 9]

How it works

  • Checks args.length against fn.length (arity)
  • If enough args → call original function
  • Otherwise → return a closure accumulating more args
  • Uses recursion via curried.apply

Currying vs Partial Application

Currying: transforms a function of N args into N unary functions.
Partial application: fixes some arguments upfront via bind or closures. Currying enables partial application automatically.

Q11 — Implement Memoization

The Question

Implement a generic memoize(fn) that caches results. Then write a memoized Fibonacci function. What are the memory trade-offs?

// Generic memoize
function memoize(fn) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) {
      console.log('cache hit:', key);
      return cache.get(key);
    }
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

// Memoized Fibonacci
const fib = memoize(function(n) {
  if (n <= 1) return n;
  return fib(n - 1) + fib(n - 2);
});

console.log(fib(40)); // instant — O(n) not O(2^n)

// LRU Cache variant (bounded memory)
function memoizeLRU(fn, maxSize = 100) {
  const cache = new Map();
  return function(...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) {
      const val = cache.get(key);
      cache.delete(key); cache.set(key, val); // reorder
      return val;
    }
    if (cache.size >= maxSize)
      cache.delete(cache.keys().next().value); // evict oldest
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

Trade-offs

  • Pros: Massive speedup for expensive pure functions
  • Cons: Memory grows unbounded (use LRU to cap)
  • JSON.stringify key fails for functions, circular refs
  • Only suitable for pure functions (same input → same output)

When to Use

  • Fibonacci, factorial, combinatorics
  • Expensive DOM measurements
  • API response caching
  • Selector memoization (like createSelector in NgRx)

Q12 — Implement Deep Clone

The Question

Write a deepClone(obj) that handles nested objects, arrays, Date, RegExp, and circular references. Compare approaches.

function deepClone(obj, visited = new WeakMap()) {
  if (obj === null || typeof obj !== 'object') return obj;

  // Handle circular references
  if (visited.has(obj)) return visited.get(obj);

  // Special types
  if (obj instanceof Date)   return new Date(obj);
  if (obj instanceof RegExp) return new RegExp(obj);
  if (obj instanceof Map) {
    const m = new Map();
    visited.set(obj, m);
    obj.forEach((v, k) => m.set(deepClone(k, visited),
                                    deepClone(v, visited)));
    return m;
  }

  const clone = Array.isArray(obj) ? [] : {};
  visited.set(obj, clone);

  for (const key of Object.keys(obj)) {
    clone[key] = deepClone(obj[key], visited);
  }
  return clone;
}

Q13 — Flatten a Deeply Nested Array

The Question

Implement flatten(arr, depth) without using Array.prototype.flat. Provide 3 different approaches: recursive, iterative (stack), and reduce-based.

const input = [1, [2, [3, [4]], 5]];

// 1. Recursive — infinite depth
function flattenRecursive(arr) {
  return arr.reduce((acc, item) =>
    acc.concat(Array.isArray(item) ? flattenRecursive(item) : item), []);
}

// 2. Iterative stack — avoids call stack overflow
function flattenStack(arr) {
  const stack = [...arr];
  const result = [];
  while (stack.length) {
    const item = stack.shift();
    if (Array.isArray(item)) stack.unshift(...item);
    else result.push(item);
  }
  return result;
}

// 3. With depth control
function flatten(arr, depth = Infinity) {
  if (depth === 0) return [...arr];
  return arr.reduce((acc, item) =>
    acc.concat(Array.isArray(item) ? flatten(item, depth - 1) : item), []);
}

flatten(input);     // [1, 2, 3, 4, 5]
flatten(input, 1); // [1, 2, [3, [4]], 5]

// Native (ES2019)
input.flat(Infinity);  // [1, 2, 3, 4, 5]

Performance Notes

  • Recursive: simple, but risks stack overflow for very deep nesting
  • Iterative stack: O(n) space, safer for large data
  • Native .flat(): fastest — use in production

Related: flatMap

[1,2,3].flatMap(x => [x, x * 2])
// [1,2, 2,4, 3,6]

Equivalent to .map().flat(1). Useful for expanding each item into multiple results.

Array.prototype.myMap = function(cb) {

Q15 — Implement Promise.all & Promise.allSettled

The Question

Implement myPromiseAll(promises) and myPromiseAllSettled(promises) from scratch. This is a very common senior-level question.

function myPromiseAll(promises) {
  return new Promise((resolve, reject) => {
    if (!promises.length) return resolve([]);
    const results = [];
    let pending = promises.length;

    promises.forEach((p, i) => {
      Promise.resolve(p)
        .then(val => {
          results[i] = val;          // preserve order
          if (--pending === 0) resolve(results);
        })
        .catch(reject);               // first failure → reject all
    });
  });
}

function myPromiseAllSettled(promises) {
  return new Promise(resolve => {
    if (!promises.length) return resolve([]);
    const results = [];
    let pending = promises.length;

    promises.forEach((p, i) => {
      Promise.resolve(p)
        .then(value => {
          results[i] = { status: 'fulfilled', value };
          if (--pending === 0) resolve(results);
        })
        .catch(reason => {
          results[i] = { status: 'rejected', reason };
          if (--pending === 0) resolve(results);
        });
    });
  });
}

// Bonus: Promise.race
function myPromiseRace(promises) {
  return new Promise((resolve, reject) => {
    promises.forEach(p => Promise.resolve(p).then(resolve, reject));
  });
}

Key Implementation Details

  • Wrap each in Promise.resolve(p) to handle non-Promise values
  • Track count with pending counter — don't use results.length (indices may be sparse)
  • Preserve insertion order by using index results[i]
  • For allSettled: never call reject — always resolve with status objects

Concurrency Limit (bonus)

// Run at most N promises at a time
async function poolAll(fns, limit) {
  const pool = new Set();
  for (const fn of fns) {
    const p = fn().finally(() => pool.delete(p));
    pool.add(p);
    if (pool.size >= limit) await Promise.race(pool);
  }
  return Promise.allSettled(pool);
}

Q16 — Event Delegation & Custom Event Emitter

The Question

Explain event bubbling and delegation. Then implement a simple EventEmitter class (Node.js EventEmitter-style) with on, off, emit, and once.

// Event Delegation pattern
// Instead of 1000 listeners, add ONE to the parent
document.querySelector('#list')
  .addEventListener('click', (e) => {
    const item = e.target.closest('[data-id]');
    if (item) console.log(item.dataset.id);
  });

// Custom EventEmitter
class EventEmitter {
  constructor() { this.events = {}; }

  on(event, listener) {
    (this.events[event] ??= []).push(listener);
    return this; // chainable
  }

  off(event, listener) {
    this.events[event] =
      (this.events[event] || []).filter(l => l !== listener);
    return this;
  }

  emit(event, ...args) {
    (this.events[event] || []).forEach(l => l(...args));
    return this;
  }

  once(event, listener) {
    const wrapper = (...args) => {
      listener(...args);
      this.off(event, wrapper); // self-removes
    };
    return this.on(event, wrapper);
  }
}

const bus = new EventEmitter();
bus.on('login', user => console.log('Welcome', user));
bus.once('init', () => console.log('App started'));
bus.emit('login', 'Nitin');
bus.emit('init'); // fires
bus.emit('init'); // no-op — already removed

Event Phases

  • Capturing: window → target (3rd arg: true)
  • Bubbling: target → window (default)
  • e.stopPropagation() stops bubbling
  • e.preventDefault() stops default browser action

⚡ Angular: The EventEmitter class in Angular IS a Subject (RxJS) under the hood. The pattern above is the conceptual model for how component @Output() events work.

Q17 — Generator Functions & Iterators

The Question

Implement an infinite ID generator, a range(start, end, step) generator, and explain how generators enable lazy evaluation and async control flow.

// Infinite unique ID generator
function* idGenerator(start = 1) {
  while (true) yield start++;
}
const ids = idGenerator();
ids.next().value; // 1
ids.next().value; // 2 — paused between yields

// Range generator (lazy — no array created)
function* range(start, end, step = 1) {
  for (let i = start; i < end; i += step) yield i;
}
[...range(0, 10, 2)]; // [0, 2, 4, 6, 8]

// Two-way communication
function* calculator() {
  let total = 0;
  while (true) {
    const input = yield total; // yield sends, receives
    if (input === null) return total;
    total += input;
  }
}

calc.next(10); // { value:10, done:false }
calc.next(5);  // { value:15, done:false }
calc.next(null); // { value:15, done:true }

// Async generators (for async streams)
async function* paginate(url) {

  while (true) {
    const data = await fetch(`${url}?page=${page++}`).then(r => r.json());
    if (!data.length) break;
    yield data;
  }
}

Generator Protocol

  • function* creates a generator factory
  • Calling it returns an iterator object
  • .next(val) resumes; val becomes the result of the yield expression
  • .return(v) terminates; .throw(e) injects an error

⚡ Angular/RxJS: Generators power async iterables. The from(asyncIterable) RxJS operator can wrap async generators into Observables.

Q18 — WeakMap, WeakSet & Memory Management

The Question

What is the difference between Map/Set and WeakMap/WeakSet? Give a practical use case for WeakMap that avoids memory leaks.

// Map holds strong reference — memory leak risk
const cache = new Map();
let domNode = document.querySelector('#btn');
cache.set(domNode, { clicks: 0 });
domNode = null; // ❌ Map still holds reference!

// WeakMap — GC can collect when object unreachable
const wCache = new WeakMap();
let domNode2 = document.querySelector('#btn');
wCache.set(domNode2, { clicks: 0 });
domNode2 = null; // ✅ entry auto-removed by GC

// Use case 1: Private class data
const _private = new WeakMap();
class BankAccount {
  constructor(balance) {
    _private.set(this, { balance }); // truly private
  }
  deposit(amount) {
    _private.get(this).balance += amount;
  }
  getBalance() { return _private.get(this).balance; }
}

// Use case 2: Memoization tied to object lifecycle
const memo = new WeakMap();
function expensiveCalc(obj) {
  if (memo.has(obj)) return memo.get(obj);
  const result = /* heavy computation */ obj.value * 2;
  memo.set(obj, result);
  return result;
}

// WeakSet: tracking visited nodes
const seen = new WeakSet();
function processNode(node) {
  if (seen.has(node)) return;
  seen.add(node);
  // process...
}

Q19 — Proxy & Reflect: Reactive Object

The Question

Use Proxy to create an observable object that logs every property access and mutation. Then implement input validation using a Proxy.

// Observable object via Proxy
function createReactive(target, onChange) {
  return new Proxy(target, {
    get(obj, key) {
      console.log(`GET: ${String(key)}`);
      return Reflect.get(obj, key); // forward correctly
    },
    set(obj, key, value) {
      const old = obj[key];
      const result = Reflect.set(obj, key, value);
      if (old !== value) onChange(key, old, value);
      return result; // must return true
    },
    deleteProperty(obj, key) {
      console.log(`DELETE: ${String(key)}`);
      return Reflect.deleteProperty(obj, key);
    }
  });
}

const state = createReactive(
  { count: 0 },
  (k, o, n) => console.log(`${k}: ${o} → ${n}`)
);
state.count++; // GET: count, count: 0 → 1

// Validation Proxy
function validated(target, schema) {
  return new Proxy(target, {
    set(obj, key, value) {
      if (schema[key] && !schema[key](value))
        throw new TypeError(`Invalid value for ${String(key)}: ${value}`);
      return Reflect.set(obj, key, value);
    }
  });
}

const user = validated({}, {
  age: v => Number.isInteger(v) && v > 0,
  email: v => /.+@.+\..+/.test(v)
});
user.age = 25;        // ✅ ok
user.age = 'old';     // ❌ TypeError
user.email = 'bad';  // ❌ TypeError

Available Traps

  • get, set, has (in operator)
  • deleteProperty, ownKeys
  • apply (function calls), construct (new)
  • getPrototypeOf, defineProperty

Always use Reflect

Always forward operations using Reflect.* to maintain correct semantics (this, inheritance, descriptor invariants). Returning wrong values from traps can throw TypeError.

⚡ Angular: Vue 3's reactivity system is built on Proxy. Angular uses a different model (Zone.js / signals) but understanding Proxy helps when debugging or writing framework-agnostic utilities.

// 1. Revealing Module Pattern
const ShoppingCart = (() => {
  let items = [];  // private
  const add    = item => items.push(item);
  const remove = id   => items = items.filter(i => i.id !== id);
  const total  = ()   => items.reduce((s, i) => s + i.price, 0);

// 2. Observer / Pub-Sub
class PubSub {
  constructor() { this.subs = {}; }
  subscribe(topic, fn) {
    (this.subs[topic] ??= []).push(fn);
    return () => this.unsubscribe(topic, fn); // unsubscriber
  }
  publish(topic, data) {
    (this.subs[topic] || []).forEach(fn => fn(data));
  }
  unsubscribe(topic, fn) {
    this.subs[topic] = (this.subs[topic] || []).filter(f => f !== fn);
  }
}

// 3. Singleton (module-level — JS modules are singletons)
class AppConfig {
  static #instance = null;
  #settings = {};

  static getInstance() {
    if (!AppConfig.#instance)
      AppConfig.#instance = new AppConfig();
    return AppConfig.#instance;
  }
  set(k, v) { this.#settings[k] = v; }
  get(k)    { return this.#settings[k]; }
}
