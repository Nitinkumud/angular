01 — Global Context

Global Context

When this is used outside of any function or object, it refers to the global object.

Browser

The global object is window.

// Browser
console.log(this);       // Window {...}
console.log(this === window); // true

Node.js

The global object is global, but at module scope this is {} (the module's exports).

// Node.js (CommonJS module)
console.log(this);      // {} (module.exports)
console.log(global);   // [Object: global]
💡 globalThis (ES2020) is the universal cross-environment reference to the global object — works in browser, Node.js, and workers. 
// Browser – top-level script
console.log(this);   // Window

var x = 42;
console.log(this.x); // 42 (var leaks to window)

let y = 99;
console.log(this.y); // undefined (let doesn't)

// ES2020+
console.log(globalThis); // Window (browser)
                           // global  (Node.js)
                           // self    (Worker)
⚠️ Gotcha: var at global scope attaches to window, but let and const do not.
02 — Function Context

Function Context

Behavior of this inside a regular function depends on strict mode.

Non-Strict Mode

Called a default binding. this is the global object.

function show() {
  console.log(this);
}

show(); // Window (browser)
        // global (Node.js)
Default binding: When no other rule applies, non-strict this falls back to the global object.

Strict Mode

this is undefined — prevents accidental global pollution.

"use strict";

function show() {
  console.log(this);
}

show(); // undefined

// Also applies inside class bodies (always strict)
class Foo {
  bar() { console.log(this); }
}
const fn = new Foo().bar;
fn(); // undefined (strict by default in class)
💡 All class bodies in ES6 run in strict mode automatically, so method extraction always yields undefined for this.
03 — Method Context

Method Context (Inside an Object)

When called as an object method, this refers to the object to the left of the dot (implicit binding).

const person = {
  name: 'Alice',
  greet() {
    console.log(`Hi, I'm ${this.name}`);
  }
};

person.greet(); // "Hi, I'm Alice"

// ⚠️ Method extraction LOSES context
const g = person.greet;
g(); // "Hi, I'm undefined"  (strict: TypeError)
⚠️ Implicit binding lost: Storing a method in a variable and calling it removes the object context.

Nested Object

const outer = {
  name: 'outer',
  inner: {
    name: 'inner',
    show() {
      console.log(this.name);
    }
  }
};

outer.inner.show(); // "inner"
// this = the object IMMEDIATELY left of the dot
💡 Rule: this is the object that owns the call — the one directly before the final dot.
04 — Arrow Functions

Arrow Functions & Lexical this

Arrow functions do not have their own this. They capture this from the enclosing lexical scope at definition time.

Correct Use — Preserving Context

const timer = {
  count: 0,
  start() {
    setInterval(() => {     // arrow: this = timer
      this.count++;
      console.log(this.count);
    }, 1000);
  }
};
timer.start(); // 1, 2, 3 ... ✓

Wrong Use — Arrow in Object Literal

const person = {
  name: 'Bob',
  // ❌ arrow captures `this` from the
  //    module/global scope, NOT the object
  greet: () => {
    console.log(this.name);
  }
};
person.greet(); // undefined ❌
❌ Never use arrow functions as object literal methods when you need this to refer to the object.
💡 Arrow functions also cannot be used as constructors (new ArrowFn() throws a TypeError) and cannot be re-bound with .call(), .apply(), or .bind().
05 — Event Handlers

Event Handlers

In a DOM event handler, this refers to the element that fired the event.

Regular Function Handler

// HTML: <button id="btn">Click Me</button>

document
  .getElementById('btn')
  .addEventListener('click', function() {
    console.log(this);       // <button> element
    console.log(this.id);    // "btn"
  });

// Inline handler (same rule)
// <button onclick="console.log(this)">
// logs the button element

Arrow Function Handler

// Arrow: this comes from outer scope
document
  .getElementById('btn')
  .addEventListener('click', () => {
    console.log(this); // Window! (not button)
  });

// Use event.currentTarget instead
document
  .getElementById('btn')
  .addEventListener('click', (e) => {
    console.log(e.currentTarget); // button ✓
  });
💡 When using arrow functions in event handlers (common in React-style patterns), access the element via event.currentTarget or event.target rather than relying on this.
06 — Constructor Functions

Constructor Functions & new Binding

When a function is invoked with new, JavaScript creates a brand-new object and sets this to that object — called new binding.

function Person(name, age) {
  this.name = name;  // bound to new object
  this.age  = age;
  this.greet = function() {
    return `Hi, I'm ${this.name}`;
  };
}

const alice = new Person('Alice', 30);
console.log(alice.greet());  // "Hi, I'm Alice"
console.log(alice.age);      // 30

What new does internally

  1. Creates a new empty object: {}
  2. Sets [[Prototype]] to Person.prototype
  3. Binds this to that new object
  4. Executes the constructor body
  5. Returns the new object (unless the constructor explicitly returns an object)
⚠️ If a constructor returns a plain object explicitly, this is discarded and that object is returned instead. Primitives are ignored.
07 — Class Context

Class Context

In ES6 classes, this refers to the instance. Class bodies run in strict mode automatically.

class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    return `${this.name} makes a noise.`;
  }
}

class Dog extends Animal {
  speak() {
    return `${this.name} barks.`;
  }
}

const d = new Dog('Rex');
console.log(d.speak()); // "Rex barks."

Class Fields — Arrow vs Regular

class Counter {
  count = 0;

  // Arrow: this is FIXED to instance ✓
  increment = () => { this.count++; };

  // Regular: this can be lost ⚠️
  decrement() { this.count--; }
}

const c = new Counter();
const { increment, decrement } = c;

increment(); // ✓ works — c.count = 1
decrement(); // ❌ TypeError: this is undefined
💡 Class field arrow functions are a common React pattern to avoid binding in constructors. Each instance gets its own copy of the function.
08 — Explicit Binding

call / apply / bind

These methods let you set this explicitly — called explicit binding.

call

Invokes immediately. Arguments passed individually.

function greet(city) {
  return `${this.name}, ${city}`;
}
const u = { name: 'Alice' };

greet.call(u, 'NYC');
// "Alice, NYC"
apply

Invokes immediately. Arguments passed as an array.

greet.apply(u, ['LA']);
// "Alice, LA"

// Useful for variadic spread:
Math.max.apply(null, [1,5,3]);
// 5
bind

Returns a new function with this permanently set.

const bound = greet.bind(u);
bound('SF'); // "Alice, SF"

// Partial application:
const nyc = greet.bind(u, 'NYC');
nyc(); // "Alice, NYC"
call(thisArg, arg1, arg2, ...)
Immediately invoked. Spreads args.
apply(thisArg, [args])
Immediately invoked. Array args.
bind(thisArg, ...partials)
Returns new function. Supports partial application.
09 — Binding Priority Rules ✦ Added

The 4 Rules & Their Priority

When multiple rules could apply, JavaScript follows a strict precedence order.

1. new Binding
›
2. Explicit Binding
call / apply / bind
›
3. Implicit Binding
obj.method()
›
4. Default Binding
global / undefined

bind beats call

function who() { return this.name; }
const bound = who.bind({ name: 'Alice' });

// call can't override a hard bind:
console.log(bound.call({ name: 'Bob' }));
// "Alice"  ← bind wins!

new beats bind

function Foo(x) { this.x = x; }
const BoundFoo = Foo.bind({ x: 999 });

const obj = new BoundFoo(42);
console.log(obj.x); // 42  ← new wins!

10 — ES Modules & globalThis ✦ Added

ES Modules & globalThis

Two modern behaviors that trip up experienced developers.

this at Top Level in ES Modules

<!-- index.html -->
<script type="module" src="app.js"></script>

// app.js — ES module
console.log(this);
// undefined  ← NOT window!

// Compare: classic script
<script src="app.js"></script>
console.log(this);
// Window  ← global as expected
⚠️ ES modules are implicitly in strict mode. Top-level this is undefined, not window.

globalThis — ES2020

// Works everywhere: browser, Node.js, worker
console.log(globalThis);

// Before globalThis, developers wrote:
const getGlobal = () => {
  if (typeof window !== 'undefined') return window;
  if (typeof global !== 'undefined') return global;
  if (typeof self   !== 'undefined') return self;
};

// Now just:
console.log(globalThis); // Window | global | self

this in Getters & Setters

const circle = {
  radius: 5,
  get area() {
    return Math.PI * this.radius ** 2;
  }
};
console.log(circle.area); // 78.53...
11 — Common Pitfalls

Common Pitfalls

Pitfall 1

Losing this in Callbacks

const obj = {
  name: 'Alice',
  greet() {
    setTimeout(function() {
      console.log(this.name); // ❌ undefined
    }, 100);
  }
};
obj.greet();

// Fix: use arrow function
setTimeout(() => console.log(this.name), 100); // ✓
Pitfall 2

Arrow Function as Object Method

const person = {
  name: 'Bob',
  greet: () => {
    // ❌ this is NOT person, it's outer scope
    console.log(this.name); // undefined
  }
};
person.greet();

// Fix: use shorthand method
greet() { console.log(this.name); } // ✓
Pitfall 3

Method Extraction

class Timer {
  constructor() { this.ticks = 0; }
  tick() { this.ticks++; }
}
const t = new Timer();
// ❌ setInterval gets a bare function ref
setInterval(t.tick, 1000);

// Fix options:
setInterval(() => t.tick(), 1000);        // ✓
setInterval(t.tick.bind(t), 1000);         // ✓
Pitfall 4

Nested Regular Function

const obj = {
  val: 42,
  outer() {
    function inner() {
      return this.val; // ❌ this is global
    }
    return inner();
  }
};
console.log(obj.outer()); // undefined

// Fix: arrow function
const inner = () => this.val; // ✓
12 — Interview Questions (Part 1)

Output-Based Questions Q1 – Q5

Q1 — Method Extraction
const person = { name: 'Alice', getName() { return this.name; } };
const fn = person.getName;
console.log(fn());
Output
undefined (strict mode: TypeError: Cannot read properties of undefined)
Reason: fn is called without a receiver — default binding applies.
Q2 — bind vs call
function greet() { return this.name; }
const bound = greet.bind({ name: 'Alice' });
console.log(bound.call({ name: 'Bob' }));
Output
"Alice"
Reason: bind creates a hard binding. Calling .call() on a bound function cannot override the bound this.
Q3 — Arrow in setTimeout
const obj = {
  name: 'Alice',
  greet() {
    setTimeout(function() { console.log(this.name); }, 0);
  }
};
obj.greet();
Output
undefined (browser non-strict) / TypeError (strict)
Reason: The setTimeout callback is a regular function. Its this defaults to window (or is undefined in strict mode).
Q4 — new beats bind
function Foo(val) { this.val = val; }
const Bound = Foo.bind({ val: 999 });
const obj = new Bound(42);
console.log(obj.val);
Output
42
Reason: new has higher priority than bind. The bound object { val: 999 } is ignored; a fresh instance is created.
Q5 — Nested Object Access
const a = { x: 1, b: { x: 2, get() { return this.x; } } };
console.log(a.b.get());
Output
2
Reason: this = the object immediately to the left of the last dot = b. So this.x = 2.
12 — Interview Questions (Part 2)

Output-Based Questions Q6 – Q10

Q6 — Class Field Arrow
class Counter {
  count = 0;
  increment = () => { this.count++; };
  decrement() { this.count--; }
}
const c = new Counter();
const { increment, decrement } = c;
increment();
console.log(c.count);
Output
1 (then calling decrement() would throw TypeError because this is undefined)
Reason: Arrow class field has this permanently bound to the instance. Regular method loses it on extraction.
Q7 — Prototype Method Extraction
function Animal(name) { this.name = name; }
Animal.prototype.speak = function() { return this.name; };
const dog = new Animal('Rex');
const speak = dog.speak;
console.log(speak());
Output
undefined (strict: TypeError)
Reason: Extracted method, default binding — this is window/undefined.
Q8 — Arrow Inside Class Method
class Obj {
  val = 10;
  outer() {
    const inner = () => this.val;
    return inner();
  }
}
console.log(new Obj().outer());
Output
10
Reason: The arrow function captures this from outer()'s scope, which is the instance.
Q9 — ES Module Top-Level this
// <script type="module">
console.log(this);
console.log(this === window);
Output
undefined
false
Reason: ES modules are always in strict mode. Top-level this is undefined, not window.
Q10 — apply with null
"use strict";
function show() { console.log(this); }
show.apply(null);
show.call(undefined);
Output
null
undefined
Reason: In strict mode, this keeps exactly the value passed to call/apply/bind. In non-strict mode both would coerce to window.

Summary

Key Takeaways

Rule 1

It's all about the call site. this is determined when a function is called, not when it's defined — except for arrow functions.

Rule 2

Arrow functions inherit this lexically. They capture it from the enclosing scope at definition time and it cannot be changed.

Rule 3

Binding priority: new > explicit (bind/call/apply) > implicit (obj.fn()) > default (fn()).

Rule 4

Strict mode and ES modules set this to undefined in default binding scenarios — always prefer explicit binding.

Rule 5

Use class field arrows to permanently bind this to an instance — ideal for event callbacks and React component handlers.

Rule 6

globalThis is the safe cross-environment way to access the global object. Top-level this in an ES module is undefined.
