Strictly Typed Forms

🌍 Analogy

A regular form = plain cardboard box (anything fits). A Typed Form = labelled storage compartment — each slot has a name & type. TypeScript shows errors before you run the app if you put the wrong thing in.

4 Typed Form Classes

1

FormControl<T> — single field. FormControl<string> = always a string.

2

FormGroup<T> — group of related fields (e.g. "Personal Info" section).

3

FormArray<T> — a list of fields (e.g. multiple phone numbers).

4

FormRecord<T> — map with runtime keys (e.g. user-defined custom fields).

.value vs .getRawValue()

PropertyReturnsDisabled fields?
.valuePartial — fields may be missing❌ Excluded
.getRawValue()Complete — ALL fields present✅ Included

⚠️ Always use getRawValue() when submitting!

🧠 Memory Trick

NNFB = "Never Null Form Builder" — Use NonNullableFormBuilder for required fields. Use new FormControl<T|null>(null) only for optional/nullable fields.

login.component.ts
// 1. Define the control shape (what TYPE each field holds)
interface LoginControls {
  email:    FormControl<string>;       // NEVER null (NonNullable)
  password: FormControl<string>;       // NEVER null
  age:      FormControl<number | null>; // CAN be null (optional)
}

@Component({ standalone: true, imports: [ReactiveFormsModule], ... })
export class LoginComponent {
  private fb = inject(NonNullableFormBuilder); // NNFB

  form: FormGroup<LoginControls> = this.fb.group({
    // fb.control() = nonNullable by default (string, not string|null)
    email:    this.fb.control('', [Validators.required, Validators.email]),
    password: this.fb.control('',  Validators.required),

    // new FormControl = explicitly nullable
    age: new FormControl<number | null>(null),
  });

  onSubmit(): void {
    if (this.form.invalid) return;

    // ✅ getRawValue() → ALL fields, correct types
    const data = this.form.getRawValue();
    // data.email: string        ← not string|null
    // data.age:   number | null  ← correctly typed
    console.log('✅ Submit payload:', data);

    // ❌ .value → Partial, disabled fields MISSING
    console.log('⚠️ Partial value:', this.form.value);
  }

  // TypeScript compiler blocks wrong types at write time:
  // form.controls.email.setValue(123) ← ❌ ERROR: number ≠ string
  // form.controls.email.setValue("ok") ← ✅ fine
}

Async Validation & Performance

🌍 Analogy

Like a hotel receptionist checking room availability. You don't ask on every letter you type — you wait until you finish typing (blur). They show a "Checking…" spinner, then say ✅ Available or ❌ Taken.

6-Step Lifecycle

1

User types email

2

User tabs away (blur) — only NOW validator fires (updateOn:'blur')

3

Control → PENDING state

4

After 400ms debounce via timer() → HTTP call fires

5

If blurred again before 400ms → switchMap cancels old request

6

Response arrives → VALID or INVALID

TSF Rule — 3 Operators

OperatorPlain English
Timer(400)Wait 400ms — debounce before HTTP call
SwitchMap()Cancel previous, start fresh on new value
First()Complete stream — exits PENDING state ⚠️
🧠 Memory: TSF Rule

Timer → SwitchMap → First

Forget F = form stuck in PENDING forever! 🔄

unique-email.validator.ts + register.component.ts
// ── Functional async validator factory ──
export function uniqueEmailValidator(): AsyncValidatorFn {
  const http = inject(HttpClient); // inject inside component constructor

  return (ctrl: AbstractControl) => {
    const email = ctrl.value as string;
    if (!email) return of(null);  // empty = skip check

    return timer(400).pipe(      // T — wait 400ms
      switchMap(() =>             // S — cancel old, start new
        http.get<{ exists: boolean }>('/api/check-email', {
          params: { email }        // always use params object!
        })
      ),
      map(r => r.exists
        ? { emailTaken: true }   // error → INVALID
        : null),                    // null → VALID
      catchError(() => of(null)), // API down? allow submit
      first()                      // F — complete stream!
    );
  };
}

@Component({ standalone: true, ... })
export class RegisterComponent {
  private fb = inject(NonNullableFormBuilder);

  form = this.fb.group({
    email: this.fb.control('', {
      validators:      [Validators.required, Validators.email],
      asyncValidators: [uniqueEmailValidator()],
      updateOn:        'blur'  // fire only on blur, NOT on keystroke
    })
  });

  get emailCtrl() { return this.form.controls.email; }
}

Angular Forms + Signals (v22)

🌍 Analogy — Excel Cells

A Signal = an Excel cell. Change A1, every formula using A1 auto-updates. formValue = cell A1. isActive = a formula =IF(LEN(A1)>=2, TRUE). No subscriptions needed.

TCLR — 4 Signal Primitives

T

toSignal(obs) — convert RxJS Observable → Signal. Read synchronously like a variable.

C

computed() — derive a new signal. Auto-recalculates when dependencies change.

L

linkedSignal() — writable signal that resets when its source changes (e.g. page → 1 on new query).

R

resource() — reactive async fetch. Re-runs & auto-aborts when request signals change.

RxJS vs Signals

RxJS ObservableSignal (v22)
Read valueSubscribe (async)mySignal() (sync)
Update UINeeds Zone.jsAutomatic ✓
Derived statepipe(map())computed()
CleanupManual unsubscribeAuto via DestroyRef
🧠 Memory: TCLR

ToSignal · Computed · LinkedSignal · Resource

Think: "To Create Linked Results"

search.component.ts
@Component({ standalone: true, changeDetection: ChangeDetectionStrategy.OnPush })
export class SearchComponent {
  private fb = inject(NonNullableFormBuilder);

  // Normal reactive form control
  queryCtrl = this.fb.control('');

  // T — toSignal: bridge form valueChanges → Signal
  readonly formValue = toSignal(
    this.queryCtrl.valueChanges.pipe(debounceTime(300)),
    { initialValue: '' }   // prevents Signal<string|undefined>
  );

  // C — computed: like Excel formula, auto-updates
  readonly query    = computed(() => this.formValue().trim());
  readonly isActive = computed(() => this.query().length >= 2);

  // L — linkedSignal: page resets to 1 when query changes
  readonly page = linkedSignal(() => (this.query(), 1));

  // R — resource: reactive API call, auto-aborts old requests
  readonly results = resource({
    request: () => ({ q: this.query(), p: this.page() }),
    loader: async ({ request, abortSignal }) => {
      if (!request.q) return [];
      const r = await fetch(
        `/api/search?q=${request.q}&p=${request.p}`,
        { signal: abortSignal }  // cancels stale requests
      );
      return r.json();
    }
  });
}

Custom Form Controls — ControlValueAccessor

🌍 Analogy — Power Adapter

Angular's form system speaks one language. Your custom UI (tag chips, star-rating) speaks another. CVA is the adapter. Implement 4 methods and Angular treats your component exactly like a native <input>.

WRST — 4 Methods to Implement

W

writeValue(val) — Angular → Your component. Called on setValue(). Update your internal UI.

R

registerOnChange(fn) — Store callback. Call fn(newVal) when user changes something inside.

S

setDisabledState(bool) — Called when form.disable(). Disable your internal UI.

T

registerOnTouched(fn) — Store callback. Call fn() when user blurs your component.

📋 Register Pattern
providers: [{
  provide:     NG_VALUE_ACCESSOR,
  useExisting: forwardRef(() => YourComponent),
  multi:       true
}]

Why forwardRef? Class not fully defined yet at decorator time.

🧠 Memory: WRST

Write · Register · Set · Touched

W & S = Angular → You  |  R & T = You → Angular

tag-input.component.ts
@Component({
  selector: 'app-tag-input', standalone: true,
  imports: [FormsModule],
  changeDetection: ChangeDetectionStrategy.OnPush,
  providers: [
    { provide: NG_VALUE_ACCESSOR,
      useExisting: forwardRef(() => TagInputComponent),
      multi: true }
  ],
  template: `see HTML tab`
})
export class TagInputComponent implements ControlValueAccessor {
  // Internal state as Signals (OnPush friendly)
  tags       = signal<string[]>([]);
  isDisabled = signal(false);
  inputVal   = '';

  // CVA callbacks: Angular populates these
  private _onChange = (_: string[]) => {};
  onTouched = () => {};

  // W — Angular writes value INTO component
  writeValue(v: string[] | null): void {
    this.tags.set([...(v ?? [])]); // spread → new ref
  }
  // R — store parent's change listener
  registerOnChange(fn: any) { this._onChange = fn; }
  // T — store parent's touched listener
  registerOnTouched(fn: any) { this.onTouched = fn; }
  // S — parent form.disable() → update our UI
  setDisabledState(d: boolean) { this.isDisabled.set(d); }

  add(e: Event): void {
    e.preventDefault();
    const tag = this.inputVal.trim();
    if (!tag || this.isDisabled()) return;
    this.tags.update(t => [...t, tag]);
    this.inputVal = '';
    this._onChange(this.tags()); // ← notify parent!
  }

  remove(tag: string): void {
    this.tags.update(t => t.filter(x => x !== tag));
    this._onChange(this.tags());
  }
}

Dynamic Form Generation

🌍 Analogy — Recipe Builder

The backend sends a recipe (JSON schema). The FormFactory cooks the form. The template serves it. Change the recipe → the form changes automatically — zero manual field coding.

3 Steps

1

Define Schema — JS array describing each field (key, type, label, validators).

2

FormFactory.buildGroup(schema) — generates the FormGroup automatically.

3

@for in template — renders correct input for each field type from schema.

Data Flow

Schema[]
→
FormFactory
→
FormGroup
→
@for renders
User fills
→
getRawValue()
→
API POST
🧠 Performance Tips
  • track field.key — prevents DOM re-creation
  • emitEvent: false + 1× updateValueAndValidity() for bulk fill
  • @defer (on viewport) for heavy controls
  • OnPush on every dynamic field component
form-factory.service.ts + dynamic-form.component.ts
// ── Schema type ──
interface FieldSchema {
  key:        string;
  type:       'text' | 'email' | 'number' | 'group';
  label:      string;
  required?:  boolean;
  children?:  FieldSchema[];  // for 'group' type
}

// ── FormFactory service ──
@Injectable({ providedIn: 'root' })
export class FormFactory {
  private fb = inject(NonNullableFormBuilder);

  buildGroup(schema: FieldSchema[]): FormGroup {
    const controls: Record<string, AbstractControl> = {};

    for (const f of schema) {
      if (f.type === 'group') {
        controls[f.key] = this.buildGroup(f.children ?? []);
      } else {
        const v = f.required ? [Validators.required] : [];
        controls[f.key] = new FormControl(null, v);
      }
    }
    return this.fb.group(controls);
  }
}

// ── Dynamic Form Component ──
@Component({ standalone: true, imports: [ReactiveFormsModule],
  changeDetection: ChangeDetectionStrategy.OnPush })
export class DynamicFormComponent {
  private factory = inject(FormFactory);

  readonly schema: FieldSchema[] = [
    { key: 'name',  type: 'text',   label: 'Full Name', required: true },
    { key: 'email', type: 'email',  label: 'Email',    required: true },
    { key: 'age',   type: 'number', label: 'Age',     required: false },
  ];

  readonly form = this.factory.buildGroup(this.schema);

  submit(): void {
    if (this.form.invalid) return;
    console.log(this.form.getRawValue());
  }
}

Quick Revision — All 5 Topics

1. Typed Forms

  • Use NNFB for required fields
  • new FormControl<T|null> for optional
  • Always submit with .getRawValue()
  • .value = partial, may miss disabled fields ⚠️
  • FormRecord = dynamic runtime keys
form.getRawValue() // ALL fields ✅

2. Async Validation

  • updateOn: 'blur' — no API on keystroke
  • TSF Rule: Timer → SwitchMap → First
  • Forget first() = stuck PENDING forever ⚠️
  • Show spinner via toSignal(statusChanges)
timer → switchMap → map → catchError → first()

3. Signals + Forms (v22)

  • TCLR: ToSignal → Computed → Linked → Resource
  • toSignal(valueChanges) bridges RxJS → Signal
  • computed() = lazy, memoized derived state
  • linkedSignal() = writable, auto-resets
  • resource() = reactive fetch + auto-abort

4. CVA

  • WRST: Write · Register · Set · Touched
  • W & S = Angular → Your Component
  • R & T = You → Angular (notify change/blur)
  • NG_VALUE_ACCESSOR + forwardRef
  • Add NG_VALIDATORS for form-level validation

5. Dynamic Forms

  • Schema → FormFactory → FormGroup
  • track field.key in @for
  • Bulk patch: emitEvent:false + single updateValueAndValidity()
  • @defer for heavy controls
