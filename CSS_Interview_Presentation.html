Specificity & the Cascade — Simple Intro

🎯 Real-Life Analogy — The "Who Has More Authority?" Rule

Imagine three people all give the same employee a different instruction:

👷
Intern
"Wear blue"
= element selector p { }
👔
Manager
"Wear red"
= class selector .text { }
👑
CEO
"Wear green"
= ID selector #title { }

Employee listens to the highest authority — CEO wins. That's specificity. CSS works the same way.

🌊 What is the Cascade? — 3-step decision process

Browser follows these steps IN ORDER when two CSS rules conflict:

🥇
Step 1 — Importance
!important beats everything
Author styles beat browser defaults
🥈
Step 2 — Specificity
More specific selector wins
ID > Class > Element
🥉
Step 3 — Source Order
Same specificity?
Last rule in the file wins

🏆 Specificity Score — Think of it like a 3-digit number (A, B, C)

Count each type of selector. Higher number wins. A always beats B, B always beats C — like hundreds, tens, units.

Selector TypeWhat it matchesScoreExample
A — ID #id (1, 0, 0) #submit → score 100
B — Class / Attribute / Pseudo-class .class [type] :hover :first-child (0, 1, 0) .btn → score 010
C — Element / Pseudo-element div p span ::before (0, 0, 1) p → score 001
Universal / Combinators * > + ~ (0, 0, 0) * { } → score 000
⚠️ Inline styles & !important are the exceptions

Inline style (style="...") scores (1,0,0,0) — beats all selectors. !important ignores all specificity rules entirely and forces that value — use it only as a last resort.

💡 Even 100 classes (0,100,0) CANNOT beat a single ID (1,0,0) — A column always wins.

📋 Source Order — Last rule wins (equal specificity)

/* Both rules have same specificity (0,1,0) */
.btn { color: red; }
.btn { color: blue; }  /* ✅ wins — comes last */

/* This is why CSS file order matters! */
/* Bootstrap styles first, then your overrides */
@import 'bootstrap.css';   /* first */
@import 'my-styles.css';   /* ✅ overrides bootstrap */

⚠️ !important — The Nuclear Option

/* !important ignores ALL specificity */
#title  { color: green; }          /* (1,0,0) */
p       { color: blue !important; } /* wins! */

/* !important vs !important → back to specificity */
p       { color: blue !important; }
.text  { color: red  !important; } /* wins */
/* ❌ Avoid — creates unmaintainable CSS wars */

🆕 :is() and :where() — Modern CSS (2022+)

/* WITHOUT :is() — repetitive */
header h1, header h2, header h3,
footer h1, footer h2, footer h3  { color: red; }

/* ✅ WITH :is() — clean. Specificity = most specific arg = h1 (0,0,1) */
:is(header, footer) :is(h1, h2, h3) { color: red; }

/* :where() — SAME effect but specificity is always (0,0,0) */
/* Great for base/reset styles that anyone can override easily */
:where(h1, h2, h3) { margin: 0; }  /* score (0,0,0) — any class overrides it */

/* :not() — specificity = the argument inside */
p:not(.special) { color: grey; }  /* (0,1,1) — class .special counts */
✅ Golden Rules to remember
  • Never use #id in CSS — hard to override. Use classes only.
  • Avoid !important — use it only for utility classes like .hidden { display:none !important }
  • Keep selectors short (1–2 levels) — .card .title not .nav .sidebar .card .body .title
  • When two rules conflict and you can't figure out why — calculate (A, B, C) and the higher number wins.

Pseudo-Classes & Pseudo-Elements

🔵 Pseudo-Class — styles based on STATE or POSITION

A pseudo-class selects an element when it is in a certain state. Written with a single colon :

Think of it as: "Style this element WHEN something is true about it"

  • :hover — mouse is over it
  • :focus — keyboard-focused (input, button)
  • :active — currently being clicked
  • :checked — checkbox/radio is checked
  • :disabled — form element is disabled
  • :first-child — first child of parent
  • :last-child — last child of parent
  • :nth-child(n) — every nth child
  • :not(.class) — does NOT match selector
  • :visited — link already visited

🟣 Pseudo-Element — styles a PART of an element

A pseudo-element targets a specific part of an element or inserts virtual content. Written with a double colon ::

Think of it as: "Style this PART of the element, or CREATE a virtual part"

  • ::before — inserts content BEFORE element's content
  • ::after — inserts content AFTER element's content
  • ::first-line — first line of text
  • ::first-letter — first letter (drop-cap effect)
  • ::placeholder — input placeholder text
  • ::selection — text user has highlighted
  • ::marker — list item bullet/number
  • ::backdrop — behind a dialog element

🎯 Key difference — one colon vs two colons

Pseudo-Class : (single colon)
button:hover    { background: blue; }
input:focus     { border-color: green; }
li:first-child { font-weight: bold; }
li:nth-child(2n) { background: #f0f0f0; }
/* even rows zebra striping */

👆 Selects the REAL element when something is true

Pseudo-Element :: (double colon)
p::first-letter { font-size: 2em; }
input::placeholder { color: #aaa; }
::selection    { background: #ff0; }
/* highlighted text turns yellow */

👆 Targets a PART of the element, not the whole thing

✨ ::before & ::after — Most used pseudo-elements

/* ::before and ::after create virtual child elements — need 'content' property */
.badge {
  position: relative;
}
.badge::before {
  content: "🔥 ";     /* required — even empty string '' works */
  color: orange;
}
.badge::after {
  content: " ★";
}

/* Real-world: required field asterisk */
label.required::after {
  content: " *";
  color: red;
}

/* Real-world: tooltip with CSS only */
[data-tooltip]         { position: relative; }
[data-tooltip]::after  {
  content: attr(data-tooltip); /* reads from HTML attribute */
  position: absolute; bottom: 100%;
  background: #333; color: #fff;
  display: none;
}
[data-tooltip]:hover::after { display: block; } /* show on hover */

/* Real-world: clearfix hack (old but still seen in interviews) */
.clearfix::after {
  content: '';
  display: block;
  clear: both;
}

:nth-child() — Pattern formulas

li:nth-child(1)    { } /* first item */
li:nth-child(2n)   { } /* every even: 2,4,6 */
li:nth-child(2n+1) { } /* every odd:  1,3,5 */
li:nth-child(3n)   { } /* every 3rd: 3,6,9 */
li:nth-child(-n+3) { } /* first 3 only */

/* Shorthand aliases */
li:nth-child(even) { } /* = 2n  */
li:nth-child(odd)  { } /* = 2n+1 */
💡 Single vs Double colon — Interview Trick
  • :pseudo-class — state or position (1 colon)
  • ::pseudo-element — virtual part (2 colons)
  • Old CSS2 used single colon for both (:before) — still works but double colon is correct CSS3+
  • Specificity: pseudo-class = class level (0,1,0). Pseudo-element = element level (0,0,1)

Box Model & Positioning

Box Model

Every element is a rectangular box with 4 layers:

margin
border
padding
content (width × height)

box-sizing

/* content-box (default) */
/* width = content only */
/* total = width + padding + border */
.box {
  width: 200px;
  padding: 20px;
  border: 2px solid;
  /* TOTAL = 244px ← surprise! */
}

/* border-box (recommended) */
/* width INCLUDES padding + border */
*, *::before, *::after {
  box-sizing: border-box;
  /* total = exactly 200px ✅ */
}

Flexbox — Complete Guide

🧭 Main Axis vs Cross Axis — the most important concept

flex-direction: row (default)
1
2
3
→ Main axis = HORIZONTAL (justify-content)
↕ Cross axis = VERTICAL (align-items)
flex-direction: column
1
2
3
↕ Main axis = VERTICAL (justify-content)
→ Cross axis = HORIZONTAL (align-items)

💡 justify-content always works along the main axis. align-items always works along the cross axis.

Container Properties (parent)

.container {
  display: flex;

  /* Direction */
  flex-direction: row;          /* → items in a row */
  flex-direction: column;       /* ↓ items in a column */
  flex-direction: row-reverse;  /* ← reversed row */

  /* Wrapping */
  flex-wrap: nowrap;   /* all in one line (default) */
  flex-wrap: wrap;     /* overflow to next line ✅ */

  /* Main axis alignment */
  justify-content: flex-start;    /* |123_____| */
  justify-content: flex-end;      /* |_____123| */
  justify-content: center;        /* |__123___| */
  justify-content: space-between; /* |1___2___3| */
  justify-content: space-around;  /* |_1__2__3_| */
  justify-content: space-evenly; /* |__1__2__3__| */

  /* Cross axis alignment */
  align-items: stretch;     /* fill height (default) */
  align-items: flex-start;  /* top */
  align-items: flex-end;    /* bottom */
  align-items: center;      /* middle ✅ centering! */
  align-items: baseline;    /* align text baseline */

  gap: 16px;  /* row-gap + column-gap ✅ modern */
}

Item Properties (children)

.item {
  /* flex-grow: how much to GROW */
  flex-grow: 0;  /* won't grow (default) */
  flex-grow: 1;  /* grows to fill space */
  flex-grow: 2;  /* grows twice as much as flex-grow:1 */

  /* flex-shrink: how much to SHRINK */
  flex-shrink: 1;  /* shrinks equally (default) */
  flex-shrink: 0;  /* never shrinks — fixed size */

  /* flex-basis: starting SIZE before grow/shrink */
  flex-basis: auto;   /* use content size (default) */
  flex-basis: 200px; /* start at 200px */
  flex-basis: 0;     /* start from zero */

  /* Shorthand: flex: grow shrink basis */
  flex: 1;          /* = flex: 1 1 0% ← equal share ✅ */
  flex: none;      /* = flex: 0 0 auto ← fixed size */
  flex: 0 0 150px; /* fixed 150px, no grow/shrink */

  /* Per-item overrides */
  align-self: center;   /* override parent's align-items */
  order: -1;             /* move to front visually */
}

CSS Grid — Complete Guide

Container Properties

.grid {
  display: grid;

  /* Define columns */
  grid-template-columns: 200px 1fr 1fr;    /* 3 cols */
  grid-template-columns: repeat(3, 1fr);   /* 3 equal */
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr)); /* responsive ✅ */

  /* Define rows */
  grid-template-rows: auto 1fr auto;   /* header, main, footer */

  gap: 16px;                 /* row + column gap */
  column-gap: 24px;         /* column gap only */
  row-gap: 12px;            /* row gap only */

  justify-items: stretch;  /* horizontal align of items in cell */
  align-items: stretch;    /* vertical align of items in cell */
}

Item Placement

.item {
  /* Span across columns / rows */
  grid-column: 1 / 3;      /* col line 1 to 3 (2 cols) */
  grid-column: 1 / -1;     /* full width ✅ */
  grid-column: span 2;     /* span 2 columns */

  grid-row: 1 / 3;         /* row line 1 to 3 */

  /* Named placement */
  grid-area: header;        /* must match template-areas name */

  justify-self: center;    /* override justify-items for this item */
  align-self: end;          /* override align-items for this item */
}

grid-template-areas — visual layout

.layout {
  display: grid;
  grid-template-columns: 220px 1fr;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header"
    "sidebar main  "
    "footer  footer";
  min-height: 100vh;
}
header  { grid-area: header; }
aside   { grid-area: sidebar; }
main    { grid-area: main; }
footer  { grid-area: footer; }

/* ✅ Responsive card grid — auto layout without media queries */
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 20px;
  /* ← creates as many columns as fit, each min 280px, max 1fr */
}

Flexbox vs CSS Grid — When to Use Which

Flexbox — 1D layout

  • Works in one direction at a time — row or column
  • Content-driven — items determine their own size
  • Ideal for components (nav bar, button group, card header)

Use when:

  • ✅ Navigation / toolbar
  • ✅ Centering one element
  • ✅ Row of buttons or icons
  • ✅ Sidebar + main content side by side
  • ✅ Wrapping tag chips

CSS Grid — 2D layout

  • Works in two directions — rows AND columns together
  • Layout-driven — parent defines the grid, children placed in it
  • Ideal for page layouts and complex grids

Use when:

  • ✅ Full page layout (header/sidebar/main/footer)
  • ✅ Card grid with consistent row height
  • ✅ Dashboard with tiles
  • ✅ Magazine / article layout
  • ✅ Any 2D positioning needed

Side-by-side comparison

FlexboxCSS Grid
Dimensions1D — row OR column2D — rows AND columns
ControlChildren control their sizeParent controls the grid
AlignmentAlong one axis primarilyBoth axes independently
Overlapping items❌ Not supported✅ Items can overlap using grid-area
Browser supportAll browsers ✅All modern browsers ✅
Best forComponents, nav, button groupsPage layouts, dashboards, galleries
✅ They work great TOGETHER

Use Grid for the overall page layout, then Flexbox inside each grid area to arrange the content within it. They are complementary, not competing.

CSS Custom Properties (Variables)

Defining & Using Variables

/* Define on :root — globally available */
:root {
  --primary: #3b82f6;
  --accent:  #f97316;
  --text:    #1f2937;
  --radius:  8px;
  --spacing-md: 16px;
}

/* Use with var() */
.btn {
  background: var(--primary);
  border-radius: var(--radius);
  padding: var(--spacing-md);
  /* fallback value */
  color: var(--text, #000);
}

Dark / Light Theme Toggle

:root {
  --bg:   #ffffff;
  --text: #111111;
}
[data-theme="dark"] {
  --bg:   #0d1117;
  --text: #e6edf3;
}
/* OS-level preference */
@media (prefers-color-scheme: dark) {
  :root { --bg: #0d1117; }
}

/* Change theme via JS */
document.documentElement
  .setAttribute('data-theme', 'dark');

CSS Variables in Angular — :host & ::ng-deep

/* component.scss — Angular ViewEncapsulation */
:host {
  /* Styles the component's HOST element itself */
  display: block;
  --card-radius: 8px;   /* scoped CSS variable */
}
:host(.active) { border: 2px solid blue; }   /* conditional host class */
:host-context(.dark-theme) { background: #000; } /* parent context */

/* ::ng-deep — pierce through ViewEncapsulation to style child components */
/* DEPRECATED but still used — prefer CSS variables instead */
::ng-deep .mat-button { border-radius: 4px; }

/* ✅ Better — expose CSS variable from parent, consume in child */
.parent     { --btn-radius: 4px; }
.child .btn { border-radius: var(--btn-radius, 8px); }
✅ CSS Variables vs SCSS Variables

SCSS variables ($primary) are compiled away at build time — they don't exist in the browser. CSS custom properties (--primary) are live in the browser and can be changed at runtime via JavaScript. Always prefer CSS custom properties for theming.

@media (prefers-color-scheme: dark) { }
@media (prefers-reduced-motion: reduce) { }
@media (orientation: landscape) { }
@media print { }

🆕 Container Queries (CSS 2023)

/* Respond to PARENT container size, not viewport */
/* Solves: same component in sidebar (narrow) */
/* vs main content (wide) — different layouts */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}
@container card (min-width: 400px) {
  .card {
    display: flex;         /* horizontal when wide */
  }
}
@container card (max-width: 399px) {
  .card {
    flex-direction: column; /* vertical when narrow */
  }
}

CSS Transitions & Animations

Transitions — between two states

.btn {
  background: blue;
  /* property duration timing-function delay */
  transition: background .3s ease;
  transition: all .3s ease;          /* ❌ avoid 'all' — perf */
  transition: transform .3s, opacity .2s; /* ✅ specific */
}
.btn:hover { background: darkblue; }

/* ✅ High performance: only animate these two — no reflow */
/* transform and opacity run on the GPU compositor thread */
.card       { transform: translateY(0); opacity: 1; transition: transform .3s; }
.card:hover { transform: translateY(-4px); }

Keyframe Animations

@keyframes fadeInUp {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}
@keyframes spin {
  to { transform: rotate(360deg); }
}

.element {
  /* name duration timing fill-mode delay iterations */
  animation: fadeInUp .5s ease forwards;
  animation: spin 1s linear infinite;
}

/* Respect user's motion preference */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: .01ms !important; }
}

Performance — What can be animated cheaply?

PropertyTriggersPerformance
transform (translate, scale, rotate)Composite only🟢 GPU — cheapest
opacityComposite only🟢 GPU — cheapest
background-color, colorPaint🟡 OK — no layout shift
width, height, top, leftLayout + Paint + Composite🔴 Expensive — causes reflow
✅ Golden Rule — Only animate transform & opacity

Use transform: translateX() instead of left. Use transform: scale() instead of width/height. Add will-change: transform to hint the browser to promote the element to its own GPU layer before animation starts.

  • Centering = justify-content: center + align-items: center
  • Direction changes which axis is "main"

Senior-Level Interview Q&A

Q1. What is CSS specificity? How is it calculated? What wins — class or ID?
Specificity determines which rule applies when multiple rules target the same element. It's calculated as (A, B, C): A = inline/ID selectors, B = classes/attributes/pseudo-classes, C = elements/pseudo-elements. A higher A always wins regardless of B and C. So ID (0,1,0,0) always beats class (0,0,1,0). !important overrides all specificity. Best practice: keep specificity low by using classes only, avoid IDs in CSS.
Q2. What is the difference between display:none, visibility:hidden, and opacity:0?
display:none — removes from layout entirely, no space, not accessible, no events. visibility:hidden — invisible but still occupies space in layout, children can override with visibility:visible, no events. opacity:0 — invisible, occupies space, still receives pointer events (clicks!), can be animated via GPU cheaply. Use opacity:0 + pointer-events:none for fade-out animations.
Q3. When would you use Flexbox vs CSS Grid?
Flexbox is for 1D layouts — items in a row or column. Content drives the sizing. Best for: nav bars, button groups, centering. CSS Grid is for 2D layouts — rows and columns simultaneously. Layout drives the structure. Best for: full page layout, dashboards, card grids. They work together: use Grid for the page, Flexbox inside each grid area for the content.
Q4. How does position:absolute work? What is its containing block?
An absolute element is removed from document flow and positioned relative to its nearest positioned ancestor — any ancestor with position other than static (i.e. relative, absolute, fixed, sticky). If no positioned ancestor exists, it's positioned relative to the initial containing block (viewport). Common pattern: add position:relative to a parent to contain an absolute child (tooltip, badge, dropdown).
Q5. Why should you only animate transform and opacity for performance?
Animating properties like width, top, or margin triggers the full rendering pipeline: Layout → Paint → Composite. This is expensive. transform and opacity skip Layout and Paint entirely — they only trigger the Composite step, which runs on the GPU compositor thread separate from the main thread. This results in buttery-smooth 60fps animations even when the main thread is busy. Use will-change: transform to promote the element to its own GPU layer before animation begins.
Q6. What is ::ng-deep and why is it deprecated in Angular?
::ng-deep (previously /deep/ or >>>) pierces Angular's ViewEncapsulation to style child component internals. It's deprecated because it disables encapsulation globally for the affected rule, making styles bleed unpredictably. The recommended alternatives are: (1) expose CSS custom properties (variables) as theming API of the child component, (2) use ViewEncapsulation.None if the child is intentionally unencapsulated, or (3) pass style inputs to the child component.
